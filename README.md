Instructions
You are to design and build a modern, dynamic, interactive website using SvelteKit. 

Your final mark will take into account the quality of your HTML, CSS and JavaScript, the design of the site, your use of modern web architecture, the quality of the interactivity and the deployment techniques used to make the website live.

Code quality: ensure you use semantic HTML within your Svelte components, keep your code tidy and use the correct syntax. Styling should adhere to “separation of concerns” and be compact and efficient – avoid adding unnecessary styles. Try to make your JavaScript as elegant as you can, and use variables, functions and other basic coding techniques to keep your code maintainable.

Website design: base the design of your site on inspiration you have gained from other modern, responsive websites. You don’t need to produce a unique or inventive design, rather it’s better to stick to existing best practices and established design patterns. You also need to include some features that are dynamic, i.e. different each time the user visits. Since we don’t have the infrastructure required for a back end, this will probably involve fetching data from a REST API.

Architecture: organise your code according to SvelteKit best practices, including use of modular and reusable components, component-scoped styles, sensible use of layouts and templating and correct routing practices. You’ll probably need more than two pages in your site to properly demonstrate these things. Ensure your development workflow is iterative (build the simplest version first) and that you use source code control to clearly demonstrate this.

Interaction: include user interaction to make your website dynamic. This could include features that accept user-supplied data, quizzes, games or activities. You might use JS to add interactive elements and to manipulate data, and don’t forget that Svelte comes with some nifty animations and things. Make sure your website is still secure and accessible, whatever style of interaction you include.

Deployment: make your website live on the web using current deployment practices. This should be automated and tied in with your source code control to make it easy to push fixes and new features straight to your users in a safe way.

Submission Instructions
Hand in your code in the GitHub repo provided and include the link to your deployed site in the About section in the repo main page. Include any important documentation in the README file.
